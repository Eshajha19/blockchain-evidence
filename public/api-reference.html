API Reference

Complete API Documentation for Developers

This API documentation allows developers to integrate with the EVID-DGC system. All endpoints require proper authentication.

Authentication Endpoints

POST /api/auth/login
POST /api/auth/login

Description: Login with email and password

Request Body:

{
  "email": "user@example.com",
  "password": "your_password"
}

Response:

{
  "success": true,
  "data": {
    "token": "jwt_token_here",
    "user": {
      "id": "user_id",
      "email": "user@example.com",
      "role": "investigator"
    }
  }
}

Status Codes: 200 (Success), 401 (Invalid credentials), 400 (Bad request)

POST /api/auth/wallet-connect
POST /api/auth/wallet-connect

Description: Connect MetaMask wallet for blockchain authentication

Request Body:

{
  "walletAddress": "0x...",
  "signature": "signature_from_metamask"
}

Status Codes: 200 (Success), 401 (Invalid signature), 400 (Bad request)

Case Endpoints

GET /api/cases
GET /api/cases?page=1&limit=10&status=open

Description: Get all cases (filtered by user role)

Query Parameters: page, limit, status

Response:

{
  "success": true,
  "data": {
    "cases": [{...}, {...}],
    "total": 50,
    "pages": 5
  }
}

Status Codes: 200 (Success), 401 (Unauthorized)

POST /api/cases
POST /api/cases

Description: Create new case

Request Body:

{
  "title": "Case Title",
  "description": "Case description",
  "jurisdiction": "City, State",
  "caseNumber": "2025-001"
}

Status Codes: 201 (Created), 401 (Unauthorized), 400 (Bad request)

Evidence Endpoints

POST /api/evidence/upload
POST /api/evidence/upload

Description: Upload evidence file to case

Content-Type: multipart/form-data

Form Fields: file, caseId, description, type

Response:

{
  "success": true,
  "data": {
    "evidence": {...},
    "hash": "sha256_hash",
    "blockchainTx": "transaction_id"
  }
}

Status Codes: 201 (Created), 413 (File too large), 401 (Unauthorized)

GET /api/evidence/:caseId
GET /api/evidence/:caseId

Description: Get all evidence for a case

Status Codes: 200 (Success), 404 (Not found)

Error Responses

All errors follow this format:

{
  "success": false,
  "error": "ERROR_CODE",
  "message": "Human readable error message",
  "code": 400
}

Common Error Codes:

  • 400: Bad Request - Invalid parameters
  • 401: Unauthorized - Authentication required
  • 403: Forbidden - Insufficient permissions
  • 404: Not Found - Resource doesn't exist
  • 409: Conflict - Resource already exists
  • 413: Payload Too Large - File too large
  • 500: Internal Server Error - Server error
